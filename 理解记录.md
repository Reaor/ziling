## 一、字灵是什么

**所有可见物都是由同样大小的汉字（里字）拼成的马赛克。**

- 一个里字 = 一个固定字号的汉字，占据格子网上的一个单元格
- 不存在"大字"或"小字"——原态和动态的字大小完全一样
- 不存在"单个大号渲染的巨字"——巨字是几十个里字沿着笔画轮廓排列出来的

## 二、三模态定义

### 互动态（默认）
- 进入页面即呈现
- 日程信息传给AI → AI返回反馈内容 → 动态颜文字 → 回归原态
- 再次交互 → 各种形状：花、风车、爱心、时钟、数学曲线
- 用户行为：
  - 点击 → 被点处里字散开
  - 双击 → 切换当前形状
  - 拖拽 → 壳心跟手，里字沿格子追过去（非整体平移）
  - 长按 → 回归原态
  - 多次急点 → 负面情绪颜文字
  - 长时间不点 → 自动切换形状

### 对话态
- 左下角按钮展开输入框进入；收起输入框退出
- 进入时 → 随机颜文字形状
- 用户输入 → 发给AI → AI延迟回复（期间颜文字更活跃）

**完整对话回合流转（同一批里字贯穿全程）：**

```
用户发送文字
  ↓
[等待期] 当前颜文字活跃运动
  ↓
AI返回回复（三部分结构）
  ↓
第一部分：简洁回复句
  → 当前动态里字内容更新为AI回复的字
  → 聚合成原态（文本行排列）
  ↓
用户交互（点/拖/双击）触发进入第二部分
长按则回归原态
  ↓
第二部分：巨字形态呈现
  → AI指定：什么巨字、横排竖排、多字轮番时间、持续时长
  → 里字沿格子走到巨字轮廓目标位置
  → 持续时间结束 → 进入第三部分
  → 双击 → 直接进入第三部分
  ↓
第三部分：回复流（循环输出）
  每小段 = 一句回复 + 一个颜文字表情：
    原态呈现 → 聚合成动态颜文字 → 新原态 → 新颜文字 → ...
  用户交互：
    点击 → 散开
    拖拽 → 拖动
    长按 → 回归原态
    双击 → 打断当前循环，AI从打断处重新思考并输出新回复流
  ↓
用户新输入 → 打断当前状态 → 新一轮对话（有记忆）
收起对话框 → AI继续呈现一会儿 → 自动切回互动态
```

### 游戏态
- 左上角挂件按钮进入；右上角退出按钮退出（保存进度）
- 正方形格子网，每格一个字
- 消消乐规则：点击激活一字 → 再点可组词的字 → 两个消除
- 组词判定：AI兜底（词库难穷举）
- 计分 + 补字 + 墨色溢出特效 + 音效

## 三、运动系统（三层叠加）

同一批里字，三层运动叠加决定每帧最终位置：

### 第一层：随机游走（始终运行）
- 底层引擎，每个里字独立
- 格子上走：只能横向或纵向，不能斜
- 随机选方向，一个方向上走足够多格再换方向（避免一格一换的抽搐感）
- 不碰撞：不能走进别的字占着的格子
- 速率：所有里字匀速一致

### 第二层：形状约束（形成形状时叠加）
- 形状轮廓定义了"允许活动的格子集"（掩码）
- 颜文字/巨字 → 强约束：轮廓窄，允许格子少，里字运动更有序规整
- 数学曲线/花/爱心 → 弱约束：允许区域宽松，里字运动更自由
- 里字到达目标位置后，仍在允许区域内继续随机游走

### 第三层：微动生命（颜文字模式叠加）
- 在形状约束基础上，局部区域小幅有节奏偏移
- 挤眼睛、弯眼睛、撇嘴 → 对应区域的里字目标位置做周期性 easing
- 间距呼吸感 → 全局间距做微小缩放波动

## 四、原态 ↔ 动态切换机制

**核心：始终是同一批里字，改变的是排列方式。不销毁重建。**

### 动态 → 原态
1. 当前形状里的里字 → 各自沿格子规划路径
2. 走向文本行的目标位置（按行排布，行间距 = 格子间距）
3. 到达后形成整齐的文本行
4. 过程中可能伴随内容更新（AI回复的字替换原来字的内容）

### 原态 → 动态
1. 计算目标形状的格子掩码 → 分配目标位置给每个里字
2. 里字从文本行位置 → 沿格子走到形状中的目标位置
3. 自适应增减：目标位置多 → 边缘优雅浮现新里字；目标位置少 → 多余里字优雅淡出

### 对话态流程特化
- AI返回新内容 → 当前动态里字**内容更新** → 再聚合成原态
- 不是：销毁旧的 → 生成新的 → 排成原态
- 是：**同一批里字，字符内容变了，排列方式变了，走到新位置**
- **"逐个消失→浮现"是技术妥协，非设计意图。实现了MAPF后可直接内容更新+走位过渡。**

## 五、关键技术本质

1. **多智能体路径规划（MAPF）**：100-300个智能体在离散格子上并发寻路，不碰撞
2. **形状采样 → 格点映射**：离屏Canvas渲染目标形状 → getImageData提取像素 → 映射到格点坐标 → 生成"允许格子集"掩码
3. **里字生命周期**：自适应增减——优雅浮现/淡出，不突然出现或消失
4. **格子内插值动画**：即使逻辑上是离散格子移动，视觉上需要子像素平滑过渡

## 六、技术选型结论

**Canvas 2D + Vanilla JS + WebView嵌入**

原因：
- 渲染需求极简：fillText(x, y, char) 就够
- 核心难点是算法（MAPF、形状采样），不是渲染
- 离屏Canvas做形状采样天然顺手
- 零依赖，直接 file:// 加载嵌入移动端WebView
- 任何框架都解决不了MAPF问题，只会增加抽象层

### PIBT 引擎核心参数
- 算法：Priority Inheritance with Backtracking（Okumura et al., AIJ 2022）
- 规划粒度：单步决策（下一格），非 A\* 长路径
- 碰撞保证：数学上保证零碰撞 + 零死锁（在双连通图上）
- 方向偏好：连续性 streak 奖励 → 长段纵横
- 插值：每字独立延时（0~60ms 随机偏移）+ 线性插值
- 参考实现：Kei18/pypibt（MIT License）
- 格子尺寸：16px cell，15px 字体
- 速度：200ms/tick
