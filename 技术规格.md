# 字灵技术规格

> 纯从需求推导，不参考任何历史实现。实施前以此文档为唯一技术依据。

---

## 一、技术选型

| 项目 | 选择 | 理由 |
|---|---|---|
| 渲染 | Canvas 2D | 需求极简：`fillText(x, y, char)`。离屏Canvas做形状采样天然顺手 |
| 语言 | Vanilla JS ES6+ | 零依赖。核心难点是算法不是框架 |
| 依赖 | 无 | 直接 `file://` 加载嵌入WebView |
| 嵌入 | WebView | 任何app框架（原生/iOS/Android/跨平台）都支持 |
| 字体 | LXGW WenKai subset | 开源，中文优美，WOFF2 ~2MB（subset后），本地加载 |
| 字号 | 默认16-18px | 可配置。移动端最小可读中文14px |
| DPR | 物理像素 = 逻辑像素 × devicePixelRatio | Canvas内部分辨率按DPR放大，CSS尺寸为逻辑像素 |

## 六、运动引擎

### 6.1 全局时钟
```
tickDuration = 200ms（默认，一字移动一格的时间）
所有里字统一速度：1格 / tick
accumulatedTime 累积 deltaTime，每超过 tickDuration 就 advanceOneTick()
```

### 6.2 路径规划算法：带时间槽预留的A\*

```javascript
function planPath(char, targetCol, targetRow, startTimeSlot):
  openSet = priorityQueue()
  cameFrom = {}
  
  while openSet not empty:
    current = openSet.pop()
    if current == target: 构建路径并返回
    
    for each neighbor (上下左右):
      if !isInBounds: continue
      nextTimeSlot = startTimeSlot + current.g + 1
      if isReserved(neighbor, nextTimeSlot): continue
      if isOccupied(neighbor) AND neighbor != target: continue
      
      turnPenalty = (方向改变 ? TURN_COST : 0)
      tentativeG = current.g + 1 + turnPenalty
      if tentativeG < neighbor.g: 更新cameFor,入队
  
  // 预约路径上的所有时间槽
  for each (cell, i) in path:
    reserve(charId, cell, startTimeSlot + i)
  return path
```

**关键参数**：
- `TURN_COST = 2.0`（转向惩罚，使路径偏好长段纵横）
- 曼哈顿距离作为启发式 `h(n)`
- 邻居顺序随机化，避免所有里字偏好同一方向

### 6.3 闲逛运动

```javascript
function wander(char):
  direction = random(['up','down','left','right'])
  steps = random(5, 15)  // 偏好长段
  target = 沿direction走steps格，碰边界则提前停止
  path = planPath(char, target, currentTimeSlot)
  char.path = path
  char.state = 'moving'
```

### 6.4 形状切换协调

```javascript
function transitionToShape(shapeMask):
  M = characters.length, N = shapeMask.length
  if N > M: spawnCharacters(N - M)
  if N < M: markExcessForDissolve(M - N)
  
  assignments = greedyAssign(characters, shapeMask)  // 贪心最近匹配
  
  for each (char, target) in assignments:
    char.targetGridX = target.x
    char.targetGridY = target.y
    path = planPath(char, target, currentTimeSlot)
    char.path = path
    char.state = 'moving'
```

### 6.5 碰撞避免保证
- 时间槽预约表保证：同一时刻同一格子只有一个里字
- 已预约未来格子，其他里字规划时绕开
- 闲逛模式周期性重规划

### 7.2 离屏采样管线
```javascript
function sampleShape(shapeDef, gridCols, gridRows):
  offCanvas = new OffscreenCanvas(gridCols, gridRows)
  offCtx = offCanvas.getContext('2d')
  
  switch shapeDef.type:
    'emoji':    渲染颜文字字符串（字体=gridRows*0.6，居中）
    'megachar': 渲染单个大字（字体=min(cols,rows)*0.9）
    'curve':    根据数学公式画线（lineWidth=1-2px）
    'live':     实时生成（如当前时间）
  
  imageData = offCtx.getImageData(0,0,gridCols,gridRows)
  pixels = []
  for y in 0..gridRows:
    for x in 0..gridCols:
      alpha = imageData.data[(y*gridCols+x)*4+3]
      if alpha > 128: pixels.push({x,y})
  
  // 稀疏化
  if pixels.length > MAX_TARGET_CHARS:
    pixels = sampleEveryNth(pixels, step)
  
  return { mask: pixels }
```

### 7.3 约束掩码
- **strict（强约束）**：里字只能在轮廓像素上，用于颜文字和巨字
- **moderate（中等）**：轮廓+周边1-2格范围，用于爱心等
- **loose（弱约束）**：轮廓+周边3-5格范围，用于曲线和时钟

## 九、渲染管线

### 9.1 每帧流程
```javascript
function render():
  ctx.clearRect(0, 0, canvas.width, canvas.height)
  if debugMode: drawGridLines()
  
  // 批量设置
  ctx.font = currentFont
  ctx.textBaseline = 'top'
  
  let prevAlpha = -1, prevColor = ''
  for char in visibleChars:
    if char.alpha != prevAlpha:
      ctx.globalAlpha = char.alpha; prevAlpha = char.alpha
    if char.color != prevColor:
      ctx.fillStyle = char.color; prevColor = char.color
    
    x = char.displayX + char.microOffsetX
    y = char.displayY + char.microOffsetY + fontSize
    ctx.fillText(char.char, x, y)
  ctx.globalAlpha = 1
```

```javascript
function initCanvas():

  cssWidth  = container.clientWidth
  cssHeight = container.clientHeight

  canvas.height = cssHeight * devicePixelRatio
```

## 十四、性能铁律——零容忍卡顿

### 14.1 帧预算
```
60fps -> 每帧上限 16.6ms
  - 运动更新（tick步进 + 插值）    <= 2ms
  - 渲染（clearRect + N x fillText） <= 10ms
  - 路径规划（跨帧分摊）           0ms（不在主帧内做）
  - 余量                           >= 4.6ms
```

### 14.2 路径规划跨帧分摊
**绝对禁止在同一帧内规划超过 5 条路径。**
```javascript
pathPlanningQueue = []
pathPlanningBudget = 5
function processPathPlanning():
  count = 0
  while queue.length > 0 AND count < budget:
    task = queue.shift()
    task.char.path = AStar(task.char, task.target)
    count++
```
形状切换时300个里字 -> 排队，每帧5条，60帧（1秒）内全部规划完毕。

### 14.3 渲染优化
- 批量设置ctx状态：font只设一次，alpha/color变化时才切换
- 跳过透明/屏幕外的里字
- 脏矩形优化留到Phase后期

### 14.4 零 GC 压力
**渲染循环内禁止创建新对象。**
```javascript
// 禁止: ctx.fillStyle = `rgb(${r},${g},${b})`
// 正确: char.colorHex预计算，直接复用
```
需要临时数组 -> 复用外部buffer。

### 14.5 字体加载——渲染前必须就绪
```javascript
async function ensureFontReady():
  font = new FontFace(FONT_FAMILY, `url(${FONT_PATH})`)
  await font.load()
  document.fonts.add(font)
ensureFontReady().then(() => startAnimationLoop())
```

### 14.6 页面可见性处理
```javascript
document.addEventListener('visibilitychange', () => {
  if (document.hidden): pauseAnimation()
  else: resumeAnimation()  // 清除累积时间差
})
```

### 14.7 Tick 步进——防范帧追赶
```javascript
const MAX_TICKS_PER_FRAME = 3
function tick(deltaTime):
  accumulatedTime += deltaTime
  ticks = 0
  while accumulatedTime >= tickDuration AND ticks < MAX_TICKS_PER_FRAME:
    advanceOneTick()
    accumulatedTime -= tickDuration
    ticks++
  if accumulatedTime >= tickDuration:
    accumulatedTime = 0  // 丢弃，不追赶
```

### 14.8 性能监控
```javascript
class PerfMonitor {
  frames = []
  measure(dt):
    this.frames.push(dt)
    if this.frames.length > 60: this.frames.shift()
    avg = average(this.frames)
    return { fps: 1000/avg, dropped: this.frames.filter(t=>t>20).length }
}
```

### 14.9 低端设备降级
```
检测：前30帧渲染耗时
if avgFrameTime > 12ms:
  MAX_CHARACTERS = 150, TICK_DURATION = 300, PATH_BUDGET = 3, DISABLE_MICRO = true
if avgFrameTime > 20ms:
  MAX_CHARACTERS = 80, TICK_DURATION = 400, DISABLE_BREATH = true
```

### 14.10 设计铁律
> **宁可少画几个字、少算几条路径，也绝不允许掉帧。**
> 16.6ms 是硬上限。任何优化砍功能不砍帧率。

### 15.2 拖拽目标
```javascript

  shellCenterX = fingerGridX
  shellCenterY = fingerGridY

    // 原形状中的相对偏移 -> 新目标 = 壳心 + 偏移
```

### 15.3 点击散开目标
```javascript
onTap(tapGridX, tapGridY):
  // 找曼哈顿距离 <= 3格的里字
  affected = characters.filter(c => manhattan(c, tap) <= 3)
  
  for char in affected:
    dx = char.gridX - tapGridX
    dy = char.gridY - tapGridY
    // 沿绝对值大的方向散开 5-8 格
    if abs(dx) > abs(dy):
      dirX = sign(dx), dirY = 0
    else:
      dirX = 0, dirY = sign(dy)
    char.targetGridX = char.gridX + dirX * random(5, 8)
    char.targetGridY = char.gridY + dirY * random(5, 8)
  
  schedulePathPlanning(affected)
  setTimeout(() => restoreCharacters(affected), 2500)  // 2.5秒后恢复
```

## 二十三、文本行布局算法

### 23.1 布局规则
```javascript
function computeTextLayout(text, gridCols, gridRows):
  chars = text split into characters  // 按字拆分（中文一字一占位）
  maxCharsPerLine = gridCols - 4      // 左右各留2格边距
  lineSpacing = 1                      // 行间空1格
  
  layout = []
  line = 0, col = 2                    // 从第2列开始（左边距）
  
  for char in chars:
    if col >= maxCharsPerLine:
      line++, col = 2
    layout.push({ col, row: 4 + line * (1 + lineSpacing) })  // 从第4行开始
    col++
  
  return layout
```

### 23.2 多行溢出
- 如果内容超过可用行数，只保留最后N行（滚动显示）
- 或者自适应增加里字数量来容纳完整内容

## 二十四、原态微动态

> L28: "对话态的原态可以增加字的稍微的动态增加活泼性"

### 24.1 实现
```javascript
function applyTextLineMicroMotion(char, time):
  if char.state != 'idle': return  // 只在静止状态下微动
  
  // 极小幅随机抖动（1-2px逻辑像素）
  seed = char.id * 12345.6789
  offsetX = sin(time * 0.8 + seed) * 1.5   // 慢速，小幅度
  offsetY = cos(time * 0.7 + seed) * 1.5
  
  char.microOffsetX = offsetX
  char.microOffsetY = offsetY
```

### 24.2 与动态游走的区别
| | 原态微动态 | 动态随机游走 |
|---|---|---|
| 幅度 | 1-2px（亚格子） | 5-15格 |
| 频率 | 慢而柔 | 正常tick驱动 |
| 格子位移 | 无（纯视觉偏移） | 有（改变gridX/Y） |
| 碰撞检测 | 不需要 | 需要 |

## 二十六、过渡中间态视觉策略

> 形状切换过程中（路径规划跨帧分摊），部分里字到目标、部分还在走、部分还没开始规划。这个"半成品"状态需要策略。

### 26.1 策略：先到等待 + 原地微动
```javascript
// 到达目标的里字：
char.state = 'arrived'
char.waiting = true
// 显示正常但做原地微动（类似原态微动态），暗示"我在等其他人"
applyTextLineMicroMotion(char, time)

// 还没开始规划的里字：
char.state = 'pending_plan'
// 原地微微上下浮动，透明显得"待命中"
char.alpha = 0.6

// 所有里字就位后：
onAllArrived():
  for char in characters:
    char.alpha = 1
    char.state = 'idle'
    // 进入形状内的闲逛模式
```

### 26.2 备选策略：统一启动时间槽
- 所有路径规划的目标起始时间槽设为 currentTimeSlot + PLAN_WINDOW（如60）
- 规划完成前里字保持当前运动
- 时间槽到达时，所有里字一起开始沿路径移动
- 优点：中间态极短；缺点：规划延迟累积

**推荐策略：统一启动（26.2）用于形状切换，先到微动（26.1）用于对话态的小幅度过渡。**
